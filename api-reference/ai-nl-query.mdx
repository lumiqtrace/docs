The Natural Language Query endpoint lets you interrogate your LLM observability data using plain English questions instead of writing SQL. You send a question, and LumiqTrace converts it to a database query, executes it safely against your project's data, and returns both the raw rows and a natural language summary of the findings. This endpoint is available on **Team plans and above**.

<Note>
Team plans include 50 NL queries per month. Scale plans have unlimited queries.
</Note>

## Endpoint

```
POST https://api.lumiqtrace.com/v1/ai/nl-query
```

**Authentication:** `Authorization: Bearer <token>` (see [Authentication](/api-reference/authentication))
**Plan:** Team+
**Rate limit:** 20 requests/hour per organization

## Request Body

<ParamField body="projectId" type="string" required>
  The ID of the project to query.
</ParamField>

<ParamField body="question" type="string" required>
  Your question in plain English. Be specific about the time range, models, or metrics you care about for the most useful results. Example: `"Which model had the highest error rate last week?"`
</ParamField>

## Example Request

<CodeGroup>
```bash curl
curl -X POST https://api.lumiqtrace.com/v1/ai/nl-query \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json" \
  -d '{
    "projectId": "proj_abc123",
    "question": "Which model had the highest error rate last week?"
  }'
```

```typescript TypeScript
const response = await fetch("https://api.lumiqtrace.com/v1/ai/nl-query", {
  method: "POST",
  headers: {
    Authorization: "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    projectId: "proj_abc123",
    question: "Which model had the highest error rate last week?",
  }),
});

const result = await response.json();
```
</CodeGroup>

## Response

**200 OK**

```json
{
  "sql": "SELECT model, countIf(status != 'success') / count() AS error_rate, count() AS total_calls FROM llm_events WHERE project_id = 'proj_abc123' AND timestamp >= now() - INTERVAL 7 DAY GROUP BY model ORDER BY error_rate DESC LIMIT 1000",
  "explanation": "This query counts the fraction of non-success events (errors, timeouts, rate-limited, cancelled) for each model over the past 7 days, then sorts models by that error rate in descending order.",
  "rows": [
    { "model": "claude-sonnet-4-6", "error_rate": 0.0421, "total_calls": 34200 },
    { "model": "gpt-4o", "error_rate": 0.0189, "total_calls": 89400 },
    { "model": "gemini-2.5-flash", "error_rate": 0.0071, "total_calls": 12800 }
  ],
  "summary": "claude-sonnet-4-6 had the highest error rate last week at 4.21% across 34,200 calls, more than double gpt-4o's rate of 1.89%. Gemini 2.5 Flash was the most reliable at 0.71%."
}
```

<ResponseField name="sql" type="string">
  The SQL query generated from your question. Useful for auditing or adapting the query for your own tooling.
</ResponseField>

<ResponseField name="explanation" type="string">
  A plain English explanation of what the SQL query does and how it answers your question.
</ResponseField>

<ResponseField name="rows" type="any[]">
  The raw query results as an array of row objects. Column names match the aliases used in the SQL query.
</ResponseField>

<ResponseField name="summary" type="string">
  An AI-generated narrative summary of the query results, interpreting the data in the context of your original question.
</ResponseField>

## Safety and Data Isolation

All generated SQL is sanitized before execution. The following SQL keywords are blocked: `DROP`, `DELETE`, `UPDATE`, `INSERT`, `CREATE`, `ALTER`, and `UNION`. Your `project_id` is automatically enforced as a `WHERE` clause condition, so queries can only access data belonging to your project. Results are capped at 1,000 rows.
