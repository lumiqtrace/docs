The traces endpoint retrieves every span belonging to a single trace and returns them as a nested tree. This is the same data structure that powers the Trace Viewer in the LumiqTrace dashboard. Use it to programmatically inspect multi-step LLM pipelines, identify which span caused a failure, or measure latency across each step in a chain.

## Endpoint

```
GET https://api.lumiqtrace.com/v1/traces/:traceId
```

**Authentication:** `Authorization: Bearer <token>` (see [Authentication](/api-reference/authentication))

## Path Parameters

<ParamField path="traceId" type="string" required>
  The `trace_id` shared by all spans in the trace you want to retrieve. This is the same value you set on events when you ingested them.
</ParamField>

## Query Parameters

<ParamField query="projectId" type="string" required>
  The ID of the project that owns this trace.
</ParamField>

## Example Request

<CodeGroup>
```bash curl
curl "https://api.lumiqtrace.com/v1/traces/trace-abc?projectId=proj_abc123" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

```typescript TypeScript
const traceId = "trace-abc";
const response = await fetch(
  `https://api.lumiqtrace.com/v1/traces/${traceId}?projectId=proj_abc123`,
  {
    headers: {
      Authorization: "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    },
  }
);

const trace = await response.json();
```
</CodeGroup>

<ResponseField name="span_id" type="string">
  Unique identifier for this span within the trace.
</ResponseField>

<ResponseField name="parent_span_id" type="string | null">
  The `span_id` of the parent span. `null` for root spans.
</ResponseField>

<ResponseField name="children" type="Span[]">
  Recursively nested child spans. An empty array means this span is a leaf node.
</ResponseField>

<Note>
All other fields on each span match the [LumiqEvent schema](/api-reference/ingest#event-schema). The only addition is the `children` array, which is not present in the raw event.
</Note>

## Understanding the Span Tree

Root spans are LLM calls that were not triggered by another LLM call — typically the first call in a pipeline. Child spans are calls that occurred inside the context of a parent call, such as an embedding lookup performed during a RAG retrieval step.

If a span in your application is not appearing as a child of the expected parent, verify that its `parent_span_id` is set to the correct `span_id` and that both spans share the same `trace_id`.
